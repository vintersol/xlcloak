  <action>
Create the token engine with two classes:

1. **TokenFormatter** in `src/xlcloak/token_engine.py`:
   - A class with a single method `format(entity_type: EntityType, counter: int) -> str`
   - Dispatch on entity_type using match statement (Python 3.10+):
     - `EntityType.EMAIL` -> `f"EMAIL_{counter:03d}@example.com"` (per D-04)
     - `EntityType.PHONE` -> `f"+10-000-000-{counter:03d}"` (per D-05)
     - `EntityType.URL` -> `f"https://example.com/URL_{counter:03d}"` (per D-07)
     - `EntityType.PERSON` -> `f"PERSON_{counter:03d}"` (per D-06)
     - `EntityType.ORG` -> `f"ORG_{counter:03d}"` (per D-06)
     - `EntityType.SSN_SE` -> `f"1000000-{counter:04d}"` (per D-08, research recommendation)
     - `EntityType.ORGNUM_SE` -> `f"000000-{counter:04d}"` (per D-08, research recommendation)
   - Raise `ValueError` if counter > 999 (per research: counter overflow at 999)

2. **TokenRegistry** in `src/xlcloak/token_engine.py`:
   - `__init__(self)`: initialize `_forward: dict[str, str] = {}`, `_reverse: dict[str, str] = {}`, `_counter: int = 0`, `_formatter = TokenFormatter()`
   - `get_or_create(self, value: str, entity_type: EntityType) -> str`:
     - If `value` already in `_forward`, return existing token (determinism per D-02 / TOK-01)
     - Otherwise: increment `_counter`, call `_formatter.format(entity_type, _counter)`, store in both maps, return token
   - `reverse_lookup(self, token: str) -> str | None`: return `_reverse.get(token)`
   - `__len__(self) -> int`: return `len(self._forward)`
   - `@property forward_map(self) -> dict[str, str]`: return copy of `_forward`
   - `@property reverse_map(self) -> dict[str, str]`: return copy of `_reverse`
