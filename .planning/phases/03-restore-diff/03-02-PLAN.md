    Add `diff` command to `src/xlcloak/cli.py` (per D-05, D-06, D-07):

    ```python
    @main.command()
    @click.argument("file", type=click.Path(exists=True, path_type=Path))
    @click.option("--bundle", "bundle_path", required=True,
                  type=click.Path(exists=True, path_type=Path),
                  help="Path to the .xlcloak restore bundle")
    @click.option("--password", default=DEFAULT_PASSWORD, show_default=True,
                  help="Decryption password for the bundle")
    @click.option("--verbose", is_flag=True, default=False,
                  help="Also show unchanged token cells and new cells")
    def diff(file, bundle_path, password, verbose):
        """Show what changed between a sanitized FILE and its BUNDLE — no files written."""
    ```

    Inside the diff command body:
    1. `BundleReader(password).read(bundle_path)` -> get `reverse_map`
    2. `WorkbookReader(file).open()` -> `iter_text_cells(wb)`
    3. Walk all text cells and classify:
       - `unchanged_tokens`: list of cells whose value is in reverse_map (token still there)
       - `changed_cells`: list of cells whose value is NOT in reverse_map, but the cell's value looks like it COULD have been a token that was modified. Actually, same problem as restore — we don't have per-cell positions.
       
       APPROACH: Like restore, track which tokens from reverse_map are found vs. missing.
       ```python
       found_tokens: dict[str, list[CellRef]] = {}  # token -> cells containing it
       non_token_cells: list[CellRef] = []
       
       for cell in reader.iter_text_cells(wb):
           if cell.value in reverse_map:
               found_tokens.setdefault(cell.value, []).append(cell)
           else:
               non_token_cells.append(cell)
       
       missing_tokens = set(reverse_map.keys()) - set(found_tokens.keys())
       ```
       
       The "changed by AI" tokens are `missing_tokens` — tokens that were in the sanitized file but are no longer present (AI replaced them with something else).

    4. Print summary header: `"{len(missing_tokens)} token(s) changed by AI."` (or `"No tokens changed by AI."` if none)
    
    5. If `missing_tokens` is not empty, render Rich table (per D-05):
       - Columns: Token | Was (original)
       - For each token in `sorted(missing_tokens)`: row = [token, reverse_map[token]]
       - Note: We can't show "Now" column because we don't know which cell the token was in. If we want to show the current value, we'd need cell positions. For V1, show Token and Original only.
       
       ACTUALLY: We CAN try harder. The diff command can compare cell-by-cell if we assume the sanitized file structure matches what was sanitized. But the AI may have added/removed rows. Keep it simple for V1: show which tokens are missing (AI changed those cells) and which are still present.
       
       REVISED TABLE (per D-05 "Sheet | Cell | Was (token) | Now"):
       Since we can't map tokens to cells without position data, adjust the table to show:
       - Column layout: Token | Original Value
       - Header: "{N} token(s) modified by AI (no longer present in file):"
       
    6. If `verbose` (per D-06): additionally show a second table "Unchanged tokens ({N}):" listing tokens still found in file with their cell locations (Sheet | Cell | Token | Original). Also show non-token cells count as a summary line.
    
    7. Footer: `click.echo(""); click.echo("No files written.")` (per D-07)
    
    8. Error handling: catch ValueError (wrong password) -> `click.echo(f"Error: {exc}", err=True)` + `sys.exit(1)`
