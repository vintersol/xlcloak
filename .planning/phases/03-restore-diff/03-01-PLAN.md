       ```python
       class Restorer:
           def __init__(self, password: str = DEFAULT_PASSWORD) -> None:

           def run(self, sanitized_path: Path, bundle_path: Path,
                   output_path: Path | None = None, force: bool = False) -> RestoreResult:
       ```
       - `run()` algorithm (per D-03, D-04):
         a. `BundleReader(self._password).read(bundle_path)` -> get `reverse_map`, `version`, `password_mode`
         b. Build a set of known token values from `reverse_map.keys()` for O(1) lookup
         c. `derive_restore_paths(sanitized_path, output_path)` -> get output paths
         d. `check_overwrite([restored_path, manifest_path], force)` (reuse from sanitizer.py, per D-09)
         e. `WorkbookReader(sanitized_path)` -> open, iter_text_cells
         f. For each text cell:
            - If `cell.value in reverse_map`: append patch `(sheet, row, col, reverse_map[cell.value])`, increment `restored_count`
            - Else if `cell.value` is NOT in reverse_map but we need to check if this cell was originally tokenized. To detect conflicts: build a `tokenized_values` set from `reverse_map.keys()`. If a cell's value is not a known token, it could be: (1) an AI-modified cell (conflict) or (2) a cell that was never tokenized. Since we don't have per-cell position tracking in the bundle, use this heuristic: a cell is "new/untouched" if its value was never a token. A cell is "skipped/conflict" only if... Actually, the bundle stores `forward_map` (original->token) and `reverse_map` (token->original). Without per-cell position data, we can only classify by VALUE: if cell value matches a known token -> restore. If not -> it's either unchanged non-PII text or AI-modified. Since we can't distinguish, classify all non-token text cells as "unchanged" (not in scope of restore). The "skipped" category applies when a cell POSITION was tokenized but value changed. Since bundle doesn't store cell positions, implement a simpler model:
              - Token match -> restore (restored_count++)
              - No token match -> leave as-is (new_count++, these are cells that either never had PII or were modified by AI)
            - IMPORTANT REVISION: The forward_map has original->token. We CAN'T tell which cell had which token without position data. So the correct approach is: walk ALL text cells in the sanitized file. If cell.value is a key in reverse_map, restore it. Otherwise, leave it. To detect "conflicts" (AI-modified cells), we need to know which cells WERE tokenized. The bundle does NOT store cell positions. So for V1, we classify: token-match=restored, non-match=unchanged. No "conflict" detection without position tracking.
            
            WAIT — re-reading the bundle: `forward_map` maps `original_text -> token`. `reverse_map` maps `token -> original_text`. The sanitized xlsx has tokens in cells. If AI changed a token cell, that cell no longer contains a token. So:
            - Cell value IS a key in `reverse_map` (it's a token) -> RESTORE to `reverse_map[cell.value]`
            - Cell value is NOT a key in `reverse_map` -> leave as-is. But HOW do we know if this was a conflict vs. never-tokenized? We CAN'T with current bundle format.
            
            SOLUTION: To enable conflict detection, during restore we also need the set of ALL token values that were placed into the sanitized file. Any token from reverse_map that does NOT appear in the current sanitized file was replaced by AI -> that's a conflict. Build it as:
            - `all_tokens = set(reverse_map.keys())`
            - `found_tokens = set()` — tokens we actually found and restored
            - After walking all cells: `missing_tokens = all_tokens - found_tokens` -> these are conflicts (tokens that were in the sanitized file but are now gone, meaning AI replaced them)
            - `skipped_count = len(missing_tokens)` but we can't say WHICH cells they were in without position data
            
            BETTER: Walk all text cells. Track which tokens from reverse_map we found. After the walk:
            - `restored_count` = number of cells where value was in reverse_map
            - `skipped_count` = `len(reverse_map) - restored_count` (tokens that should have been in file but weren't found — AI changed those cells). Note: a single original value can map to one token, so if the same token appeared in multiple cells, this needs care. Actually, forward_map is original->token (many originals could exist), reverse_map is token->original. Multiple CELLS can have the same token if they had the same original value. So counting by unique tokens found vs. total unique tokens is the right approach for the report.
            - `new_count` = total text cells walked minus cells that contained tokens
            
            For skipped_cells detail: we can't provide sheet/cell/current_value for conflicts because we don't know which cell had which token. Instead, list the MISSING TOKENS and what they would have restored to: `[{"token": "PERSON_001", "original": "John Smith"}]`.
            
            FINAL DESIGN (simpler, more honest):
            ```
            found_tokens: set[str] = set()
            patches = []
            cells_walked = 0
            
            for cell in reader.iter_text_cells(wb):
                cells_walked += 1
                if cell.value in reverse_map:
                    patches.append((cell.sheet_name, cell.row, cell.col, reverse_map[cell.value]))
                    found_tokens.add(cell.value)
            
            missing_tokens = set(reverse_map.keys()) - found_tokens
            restored_count = len(patches)  # cells restored (multiple cells can have same token)
            skipped_count = len(missing_tokens)  # unique tokens not found (AI changed those cells)
            new_count = cells_walked - restored_count  # text cells that weren't tokens
            ```
            
            For skipped_cells in report, list the missing tokens:
            ```python
            skipped_cells = [
                {"token": tok, "original": reverse_map[tok]}
                for tok in sorted(missing_tokens)
            ]
            ```

         g. Write restored xlsx via `WorkbookWriter(sanitized_path, restored_path).patch_and_save(patches)`
         h. Render and write manifest to manifest_path
